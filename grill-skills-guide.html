내 계획을
물고 늘어지는
grill 스킬

grill-me와 grill-with-docs는 모두 “내 계획을 날카롭게 질문해 허점을 찾는” 도구입니다. 차이는 하나 — 문서와 프로젝트 맥락까지 반영하느냐.

grill-me내 아이디어·계획 자체를 질문으로 검증.
grill-with-docs기존 코드·용어집·ADR과 맞는지까지 검증.
차이부터 보기 → 적용 방법
한 문장 요약

둘 다 “물고 늘어져줘”, 범위만 다르다

grill-me — “내 계획을 물고 늘어져줘.”

grill-with-docs — “내 계획을 기존 프로젝트의 언어·결정에 비춰 물고 늘어지고, 필요하면 문서까지 갱신해줘.”

비교

두 스킬의 차이

구분grill-megrill-with-docs
목적계획·설계 자체를 질문으로 검증기존 문서·코드·도메인 언어와 맞춰 검증
문서 확인필요하면 코드 탐색CONTEXT.md · docs/adr/ 확인
용어 정리가능하지만 핵심은 아님핵심 기능 — 애매한 단어를 정확한 용어로
문서 업데이트기본적으로 안 함확정 용어는 CONTEXT.md에 반영
ADR기본적으로 안 함중요한 기술 결정이면 ADR 제안 가능
추천 상황새 아이디어, 독립 계획, 문서화 전 단계기존 제품·코드베이스 안에서 변경할 때
선택

언제 무엇을 쓸까

가볍게 검증

grill-me를 쓰세요

  • 아직 프로젝트 문서와 연결하지 않아도 될 때
  • 아이디어 단계에서 허점을 찾고 싶을 때
  • 기능 설계·정책·흐름을 빠르게 점검하고 싶을 때
프로젝트 맥락 검증

grill-with-docs를 쓰세요

  • 기존 코드베이스에 기능을 추가할 때
  • 도메인 용어가 중요한 프로젝트일 때
  • 결정된 내용을 문서에 남기고 싶을 때
사용 예제

실제로 이렇게 부른다

grill-me 예제

/grill-me

새 온보딩 플로우를 만들려고 해.
사용자는 이메일 인증 후 프로필을 입력하고,
팀을 만들거나 초대 코드로 합류할 수 있어.
이 계획을 구현 전에 grill 해줘.

grill-with-docs 예제

/grill-with-docs

주문 취소 기능을 추가하려고 해.
현재는 Order 단위 취소만 있는데,
일부 상품만 취소하는 partial cancellation을 넣고 싶어.
기존 CONTEXT.md와 ADR 기준으로 grill 해줘.
필요하면 용어도 정리해줘.
비전공자용 비유

친구 vs 회사 규정집

grill-me = 친구에게 아이디어 검증받기

“이거 괜찮아?”라고 물으면 친구가 고객·가격·위험·예외 상황을 계속 되묻는 느낌입니다.

grill-with-docs = 규정집 보며 회의하기

새 아이디어를 검토하되, 기존 회사 용어·약속·과거 결정과 충돌하지 않는지 같이 확인합니다.

영상에서 강조한 좋은 사용법

이렇게 쓰면 잘 됩니다

1

범위를 작게

“전체 시스템”보다 “환불 승인 상태 전이”처럼 작게 나누세요.

2

말로 답할 것만

UI 느낌처럼 직접 봐야 아는 건 prototype으로 넘기는 편이 좋습니다.

3

방향은 내가

전부 맡기지 말고 “이건 범위 밖이야”라고 능동적으로 조정하세요.

4

결과를 버리지 말기

나온 결정은 구현·PRD·handoff·CONTEXT.md·ADR로 보존하세요.

5

똑똑한 모델로

좋은 질문을 뽑아내려면 모델의 추론력이 중요합니다.

좋은 사용 / 나쁜 사용

한 끗 차이

✓ 좋은 사용

이번 세션은 refund 상태 전이까지만 보자.
UI 느낌은 나중에 prototype으로 확인하자.

✕ 나쁜 사용

그냥 알아서 200개 질문해줘.
전체 시스템을 한 번에 완벽하게 설계해줘.

Claude Code에 넣으려면

제공 자료에는 grill 스킬의 공식 마켓 설치 명령이 확인되지 않았습니다. 같은 동작을 원하면 SKILL.md를 개인 스킬 폴더에 배치하거나, grill 지침을 CLAUDE.md에 복사해 넣습니다. 원본은 아래 “관련 링크”의 Matt Pocock 저장소를 확인하세요.

Claude Code — 개인 스킬

프로젝트/.cursor/rules/grill-with-docs.mdc
