부록 · 커뮤니티 스킬

요약본을 읽는 대신
나에게 맞춰
가르치는 teach 스킬

teach는 “이거 가르쳐줘”라고 하면, 내가 왜 배우는지(mission)에 맞춰 진도를 기억하며 짧은 인터랙티브 레슨으로 가르치는 학습 스킬입니다. 요약본을 한 번 읽고 마는 공부와는 작동 방식이 다릅니다.

그냥 요약본누구에게나 같은 내용을 한 번 읽고 끝.
teach 스킬내 목표·진도를 기억하며 다음 한 걸음을 가르침.
요약본과 뭐가 다른지 → 잘 쓰려면 필요한 것
한 문장 요약

“가르쳐줘”라고 하면, 선생님처럼 따라온다

teach — “내가 왜 이걸 배우려는지에 맞춰, 내 진도를 파일에 기억하면서, 딱 도전적인 난이도의 짧은 인터랙티브 레슨으로 가르쳐줘.”

핵심은 stateful — 폴더(워크스페이스)에 mission·진도·학습 기록을 저장해, 세션이 바뀌어도 이어서 가르칩니다. 정적인 요약 문서에는 없는 점입니다.

핵심

그냥 요약본 공부 vs teach

같은 “공부”지만, 여섯 가지가 다릅니다.

구분그냥 요약본으로 공부teach 스킬
진도 기억매번 처음부터 — 정적 문서stateful — mission·진도를 파일에 저장·누적
무엇을 가르치나누구에게나 같은 일반 내용내가 왜 배우는지(mission)에 맞춤
난이도내 수준과 무관하게 고정ZPD — 지루하지도 겁나지도 않은 딱 한 걸음
학습 방식수동적으로 읽기인터랙티브 레슨 — 퀴즈·실습·즉시 피드백
근거출처 불명 · 틀릴 수 있음고신뢰 자료에서 인용해 제시
그 다음거기서 끝knowledge→skill→wisdom, 실전·커뮤니티로 연결
ℹ
ZPD(zone of proximal development) — “근접발달영역”. 지금 내 수준에서 혼자서는 조금 버겁지만 도움받으면 해낼 수 있는 구간. teach는 매 레슨 이 구간을 노립니다.
작동 방식

폴더 하나가 학습 공간이 된다

teach는 현재 폴더를 “학습 워크스페이스”로 쓰고, 진행 상황을 파일로 남깁니다.

왜 배우는가

mission

내가 이걸 배우려는 진짜 이유를 기록. 모든 레슨이 여기에 맞춰집니다. 이게 비면 teach는 먼저 “왜 배우고 싶나”부터 묻습니다.

믿을 자료

resources

웹에서 찾은 고신뢰 자료 목록. 모델의 기억이 아니라 검증된 출처에서 지식을 끌어옵니다.

배우는 단위

lessons (HTML)

한 번에 딱 하나를 가르치는 짧은 인터랙티브 HTML. 다이어그램·퀴즈·실습이 들어가, 글 요약보다 풍부합니다. 나중에 다시 열어 복습.

진도 기록

learning records

내가 무엇을 배웠고 무엇을 이미 아는지 기록. 다음에 무엇을 가르칠지 판단하는 근거가 됩니다.

빠른 참조

reference · glossary

레슨에서 뽑은 핵심을 압축한 참고 문서(치트시트·용어집). 레슨은 한 번 보지만 이건 계속 들춰봅니다.

지식→스킬→지혜

그리고 세상으로

자료로 지식을, 인터랙티브 연습으로 스킬을, 마지막엔 커뮤니티·실전으로 보내 지혜를 얻게 합니다. 에이전트에 가둬두지 않습니다.

잘 쓰려면 필요한 것

이 다섯 가지가 있으면 잘 굴러간다

1

Claude Code(코딩 에이전트)

teach는 폴더를 워크스페이스로 쓰고 파일을 만듭니다. 파일시스템 접근이 필요해, Claude 앱·웹 단독 기능이 아니라 Claude Code 같은 코딩 에이전트에서 씁니다.

2

명확한 mission

“왜 배우려는가.” 이게 분명할수록 레슨이 내 목표에 붙습니다. 막연하면 teach가 먼저 물어봅니다.

3

전용 폴더 하나

주제마다 빈 폴더 하나를 학습 공간으로. 진도·기록·레슨이 여기 쌓입니다.

4

여러 세션 걸친 꾸준함

한 번에 끝내는 게 아니라, 짧은 레슨을 이어서 하는 stateful 학습입니다.

5

레슨에 응하는 자세

퀴즈·실습에 직접 답하고, 막히면 “이게 헷갈려”라고 되묻기. 피드백 루프가 핵심입니다.

자주 묻는 것

입문자가 자주 묻는 질문

그냥 요약 정리해달라고 하면 안 되나요?

됩니다. 다만 그건 한 번 읽고 끝입니다. teach는 내 목표·진도를 기억하고, 퀴즈·실습으로 직접 해보게 하며, 다음에 무엇을 배울지 이어줍니다.

코딩 말고 다른 것도 배울 수 있나요?

네. 원본에서도 루빅스 큐브를 배웁니다. 주제는 자유지만, 폴더에 파일을 쌓는 방식이라 코딩 에이전트(Claude Code)에서 돌아갑니다.

한 번에 다 끝나나요?

아니요. 레슨은 짧게, 하나씩입니다. 여러 세션에 걸쳐 이어가는 걸 전제로 설계됐고, 그래서 진도를 파일로 남깁니다.

내용이 틀리면요?

teach는 모델의 기억 대신 고신뢰 자료에서 인용하도록 설계됐고, 레슨에 출처 링크를 답니다. 그래도 스스로 출처를 확인하는 습관은 필요합니다.
