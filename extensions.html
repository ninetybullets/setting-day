먼저 읽기

많이 까는 게 아니라, 고르는 것

입문자가 가장 많이 하는 실수 = 처음부터 잔뜩 설치.

1빈 슬레이트로 시작

기본 에이전트를 먼저 써보고, 실제로 모자란 것만 추가하세요.

2하나씩 추가

한 번에 다 깔지 말고, 직접 고른 것을 하나씩. 무엇이 무엇을 바꾸는지 보이게.

3전역지침은 가볍게

~/.claude/CLAUDE.md는 짧게(<200줄). 길수록 따르는 정확도가 떨어집니다.

4규칙은 hook으로

"꼭 지켜야 하는 것"은 지침이 아니라 hook으로. 지침은 강제가 아닙니다.

5출처는 audit · pin

공식만 쓰고 버전을 고정. 확장은 당신 권한으로 코드를 실행합니다.

※ 아래는 공식(Anthropic)만 추린 후보입니다. 인기 커뮤니티 확장은 의도적으로 제외 — 입문자 안전 우선. 수치·이름·명령은 조사일 2026-06-23 기준, 게시 전 공식 문서 재확인 권장.

범주 ①

Skill

특화 작업용 폴더(SKILL.md). 한 번 켜면 알아서 불러 씁니다.

document-skills

공식 · 확정

문서 생성·편집 스킬 4종 묶음 — xlsx · docx · pptx · pdf.

왜 초반에: 입문자 첫 실전 작업이 대개 문서·표 생성. 범용성+즉시효용이 가장 큰 한 묶음.

① 주소 — 마켓 등록
$/plugin marketplace add anthropics/skills
② agentic / 직접 — 설치
$/plugin install document-skills@anthropic-agent-skills
⚠
주의 — 식별자 혼동: marketplace add 인자는 anthropics/skills(owner/repo)지만, install 식별자는 내부명 anthropic-agent-skills입니다. 또 이 문서 스킬은 'source-available(오픈소스 아님)' 라이선스 · 'demonstration/educational' 면책이 붙어 있습니다.

skill-creator

공식 · 확정

새 스킬을 만들고 평가하도록 단계별로 안내하는 메타 스킬.

왜 초반에: 남의 스킬을 받기보다 자기 스킬을 직접 만드는 입문 경로(원칙 2와 연결).

agentic / 직접 — 설치
$/plugin install skill-creator@claude-plugins-official

개인 스킬 직접배치

공식 · 확정방법

레지스트리 없이, SKILL.md 한 폴더면 끝. 디렉터리 이름이 곧 /명령어가 됩니다.

왜 초반에: npx/npm 불필요 — 가장 마찰이 적은 방식. 모든 프로젝트에 적용.

직접 — 폴더 만들고 SKILL.md 작성
$mkdir -p ~/.claude/skills/summarize-changes
ℹ
참고 — SKILL.md는 YAML frontmatter + 본문이면 충분하고, 필드는 전부 선택이되 description만 권장(Claude가 언제 쓸지 판단). 개인 스킬은 ~/.claude/skills/<name>/SKILL.md.

더: example-skills(예시 모음)·claude-api도 같은 마켓에서 — /plugin install example-skills@anthropic-agent-skills. (npx/npm은 skill 범주엔 해당 없음.)

범주 ②

MCP 서버

외부 도구·데이터를 Claude Code에 연결. 공식 reference 서버 7종 중 입문자용.

🔴
먼저, 아카이브 주의 — 과거 reference 서버 13개(GitHub · Slack · PostgreSQL · Google Drive · Puppeteer 등)는 아카이브됨(servers-archived, 유지보수·보안패치 중단). 옛 글에서 보여도 추천/사용 금지. 현행 reference는 아래 7종뿐입니다.

Filesystem

공식 · 확정

허용한 경로의 파일을 읽고 씁니다.

왜 초반에: 코딩 작업의 즉시효용 핵심.

npx — 경로 인자 필수
$claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /path/to/allowed/files
⚠
주의 — 허용 경로를 반드시 지정(인자 필수). 홈 전체처럼 넓게 열지 마세요.

Git

공식 · 확정

git 저장소를 읽고 조작.

왜 초반에: 버전관리는 거의 모든 개발자에게 범용.

uvx (Python) — 또는 pip install mcp-server-git
$claude mcp add git -- uvx mcp-server-git

Memory

공식 · 확정

세션 간 지식을 영속(knowledge graph)으로 보관.

왜 초반에: 도메인·선호를 기억시키는 범용 도구.

npx
$claude mcp add memory -- npx -y @modelcontextprotocol/server-memory

Fetch · Sequential Thinking · Time

현행 · 명령 미확인

Fetch(URL→마크다운) · Sequential Thinking(단계적 추론) · Time(시간대 변환) — 모두 현행 7종에 포함.

⚠
정직하게 — 이 3종은 현행 reference 서버가 맞지만, 정확한 설치 명령이 공식 메인 README에 없어 여기 추측해 적지 않습니다. 설치 전 공식 레포 modelcontextprotocol/servers의 각 서버 폴더 README에서 명령을 확인하세요.

원격(주소) MCP는 claude mcp add --transport http <name> <url> (공식 지원). 단 claude.ai/directory의 커넥터는 대부분 파트너 제작(Anthropic은 큐레이션만) — '공식 Anthropic 서버'와 구분하세요.

범주 ③

전역지침 (CLAUDE.md)

설치형이 아니라 '작성형' — 받는 게 아니라 쓰는 것.

어디에

~/.claude/CLAUDE.md = User 범위(전역, 모든 프로젝트). 로드 순서(넓음→좁음): Managed(조직) → User(전역) → Project(./CLAUDE.md) → Local(./CLAUDE.local.md).

무엇을 넣나 (전역)

  • 개인 코딩 선호, 개인 툴링 단축, "항상 X" 규칙
  • 프로젝트 고유 표준은 전역이 아니라 ./CLAUDE.md에

잘 쓰는 법 (공식 권장)

  • 파일당 200줄 미만 목표 — 길수록 따르는 정확도↓
  • 마크다운 헤더·불릿으로 구조화
  • 구체적으로: "Use 2-space indentation" > "format properly"
  • 모순되는 규칙 제거(충돌 시 임의로 하나만 따름)

적용 방법

직접 — 파일 편집
$$EDITOR ~/.claude/CLAUDE.md
agentic — Claude에게 시키기
»이 내용을 내 전역 CLAUDE.md에 추가해줘
import — 다른 파일 끌어오기 (최대 4 hops)
#@~/.claude/my-conventions.md
⚠
주의 — /init 자동생성에 의존하지 말고 핵심만 남기세요. 지침은 강제가 아니라 '맥락' — 반드시 지켜야 할 건 hook으로(원칙 4). 세션 중 빠르게 기억시키려면 # 단축 또는 /memory.
범주 ④

플러그인 / 마켓플레이스

여러 확장을 묶은 패키지. 공식 마켓 claude-plugins-official은 Claude Code 시작 시 자동 등록됩니다.

🔴
핵심 구분 — 이 공식 마켓엔 200개 넘는 플러그인이 있지만 대부분 파트너(third-party) 제작입니다. "Anthropic 공식 제작"은 author가 Anthropic인 것만 — 아래는 그중 입문자 범용 후보입니다. 모두 /plugin install <name>@claude-plugins-official.

claude-code-setup

공식 · 확정

코드베이스를 분석해 맞춤 hooks · skills · MCP · subagent를 추천.

왜 초반에: "무엇을 깔지" 자체를 안내하는 온보딩 도구 — 1일차에 가장 먼저.

$/plugin install claude-code-setup@claude-plugins-official

commit-commands

공식 · 확정

git commit · push · PR 생성 워크플로 명령.

왜 초반에: 거의 모든 개발자가 1일차부터 쓰는 범용 흐름.

$/plugin install commit-commands@claude-plugins-official

code-review

공식 · 확정

여러 전문 에이전트가 신뢰도 스코어링으로 PR을 자동 리뷰.

왜 초반에: 변경을 합치기 전 결함을 거르는 즉시효용.

$/plugin install code-review@claude-plugins-official

claude-md-management

공식 · 확정

CLAUDE.md 품질 감사 · 세션 학습 반영 · 프로젝트 메모리 최신화.

왜 초반에: 범주 ③(전역지침)을 가볍게 유지하는 짝꿍.

$/plugin install claude-md-management@claude-plugins-official

frontend-design

공식 · 확정

개성 있고 완성도 높은 프런트엔드 UI를 생성.

왜 초반에: 웹 작업에 폭넓게 쓰이는 범용 도구.

$/plugin install frontend-design@claude-plugins-official

기타 Anthropic 제작 범용 후보: feature-dev · pr-review-toolkit · plugin-dev · hookify · playground, 언어별 LSP(pyright-lsp 등). 자동등록이 안 보이면: /plugin marketplace add anthropics/claude-plugins-official. (카탈로그는 조사일 기준 일부 — 전체는 공식 문서 확인.)

커뮤니티 스킬 심화 가이드

위 큐레이션과 달리 공식(Anthropic)이 아닙니다. 입문자가 자주 묻는 커뮤니티 스킬 2종을 따로 풀어 썼습니다.

🔵
구분 — 아래 두 가이드의 스킬은 커뮤니티(비공식) 확장입니다. 위 ①~④ 공식 큐레이션과 별개로, 설치 전 원본 저장소를 직접 확인하고 버전을 고정하세요(원칙 5).
스킬 심화 커뮤니티

Karpathy Guidelines

AI 코딩 에이전트의 “안전벨트”. 과설계·범위이탈·근거 없는 ‘완료’를 줄이는 4가지 행동 원칙과 Claude Code 설치법.

가이드 열기 → 스킬 심화 커뮤니티

grill 스킬 (grill-me · grill-with-docs)

구현 전에 내 계획의 허점을 질문으로 찾는 두 스킬. 차이·사용 시점·예제와 Claude Code 적용법.

가이드 열기 →
설치 전 1분

확장 = 코드 실행. 신뢰부터.

Skill · MCP 서버 · Plugin은 당신의 권한으로 코드를 실행하고 파일·도구에 접근합니다. 공식만 골랐어도, 설치 전 3가지는 확인하세요.

① 출처 · 버전

공식(anthropics/*)인가? 버전을 고정(pin)했나? 커뮤니티 소스는 따로 audit.

② 무엇을 설치하나

설치 전 "Will install"(명령·MCP·hook) 목록을 읽었나? 모르는 건 설치하지 않기.

③ 외부 콘텐츠

민감 데이터 + 미신뢰 콘텐츠 + 외부 유출경로가 한 세션에 겹치면 위험(lethal trifecta). 분리하기.

※ 한 커뮤니티 스킬 벤치마크에선 일부 스킬이 오히려 에이전트를 악화시켰습니다 — "고르지 않은 건 깔지 않는다"가 가장 안전한 기본값.
